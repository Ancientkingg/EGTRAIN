# Writing Style

Use plain project writing.

- Keep sentences short.
- Use direct verbs.
- Name files, commands, and behavior.
- Avoid promotional language.
- Avoid filler phrases.
- Do not use em dashes.
- Do not claim impact without evidence.
- Do not mention internal tools in public docs.
- Keep headings literal.

Example:

```text
The smoke test runs Copenhagen and Brescia. It checks exit status, train movement, trajectory samples, and station-arrival output.
```
